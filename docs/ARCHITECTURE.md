# Архитектура

Проект разнесён на независимые по смыслу файлы, но оставлен без сборщика и внешних зависимостей. Это сделано намеренно: игра должна запускаться просто из `index.html` на компьютере и телефоне.

## Почему classic scripts, а не ES modules

ES modules удобны для импорта/экспорта, но локальный запуск через `file://` на части устройств и браузеров создаёт лишние ограничения. Для этой игры приоритет такой:

1. открыть ZIP;
2. открыть `index.html`;
3. игра запускается без сервера, npm и сборки.

Поэтому файлы подключаются обычными `<script defer>`. Нумерация файлов задаёт порядок загрузки.

## Важные зависимости по порядку

Порядок файлов менять нельзя без проверки:

```text
config → viewport → audio → world → flow → physics → simulation → render → ui → input/loop
```

## Физика

Базовая формула гравитации в игре:

```text
a = μ · r / |r|³
```

Где:

- `μ` — гравитационный параметр тела;
- `r` — вектор от корабля к телу;
- `|r|` — расстояние.

Физика масштабирована под игру. Это не научный симулятор в реальных километрах, но логика сохраняет ключевые принципы: масса влияет на поле, расстояние влияет по квадрату, двигатель не должен полностью перебивать гравитацию.

## Производительность

Главные правила:

- не считать гравитационную карту каждый кадр;
- ограничивать количество источников поля;
- не держать слишком много далёких тел;
- на телефоне снижать DPR и длину прогноза;
- не добавлять тяжёлые эффекты поверх минималистичного стиля.
