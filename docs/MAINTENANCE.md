# Maintenance Guide

## Перед изменениями

1. Запусти `npm run validate`.
2. Открой игру в браузере.
3. Проверь меню, старт, полёт, проигрыш, zoom, звук, карту гравитации.
4. После изменения повтори проверку.

## Что проверять вручную

### Физика

- корабль заметно изгибает траекторию рядом с планетами и звёздами;
- без тяги корабль дрейфует по инерции;
- прогноз траектории совпадает с ощущением полёта;
- двигатель не перебивает поле полностью.

### Интерфейс

- текст читается на телефоне;
- счёт при проигрыше показывается один раз;
- кнопки не перекрывают навигационную панель;
- карта гравитации не превращает экран в шум.

### Мобильная версия

- игра запускается из браузера;
- pinch zoom не ломает управление;
- кнопки внизу нажимаются;
- FPS не проваливается на старте.

## Как добавлять новый объект

1. Добавить параметры в фабрику в `04-world-generation.js`.
2. Добавить физическое влияние при необходимости в `06-physics-orbits.js`.
3. Добавить отрисовку в `08-render-space.js`.
4. Добавить понятный текст в `09-render-ui.js`, если объект участвует в задачах.

## Как добавлять новую задачу

1. Добавить тип в `buildObjective`.
2. Добавить расчёт очков в `objectiveScore` или отдельную функцию.
3. Добавить текст в `objectiveVerb`.
4. Проверить, что текст звучит естественно.

## Тон текстов

Плохо:

- «строй траекторию через гравитацию»;
- «маршрут пойман»;
- «стабилизируй связанную дугу».

Хорошо:

- «Лети от звезды к звезде. Гравитация меняет курс.»;
- «Корабль потерян»;
- «Выйди на орбиту»;
- «Дай полю разогнать корабль».

## Как собрать один HTML

```bash
npm run build:standalone
```

После этого файл появится в `dist/`.
